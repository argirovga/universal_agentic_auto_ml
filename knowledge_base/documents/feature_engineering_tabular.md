# Feature Engineering для табличных данных

## Кодирование категориальных переменных
- **Label Encoding** — подходит для деревьев (RF, GBDT). Просто нумерует категории.
- **One-Hot Encoding** — для линейных моделей. Создаёт бинарные колонки.
- **Target Encoding** — кодирует категорию средним значением целевой переменной. Осторожно: риск переобучения, используйте с кросс-валидацией.

## Обработка числовых признаков
- **Масштабирование** (StandardScaler, MinMaxScaler) — обязательно для линейных моделей, не нужно для деревьев.
- **Лог-трансформация** — для признаков с правым скосом (цены, доходы).
- **Биннинг** — разбиение непрерывных признаков на корзины. Полезно для нелинейных зависимостей.

## Создание новых признаков
- **Взаимодействия** — произведение/соотношение пар признаков.
- **Агрегации** — среднее, медиана, макс по группам (group-by features).
- **Временные признаки** — из дат: день недели, месяц, квартал, дни с события.
- **Географические** — расстояние до центра, кластеры координат.

## Отбор признаков
- Feature importance из деревьев.
- Корреляция с целевой переменной.
- Удаление мультиколлинеарных признаков (корреляция > 0.95).
