# Стратегии обработки пропущенных значений

## Анализ пропусков
1. Определить тип пропусков: MCAR (случайные), MAR (зависят от других), MNAR (зависят от самого признака).
2. Визуализировать паттерн пропусков.
3. Проверить, коррелирует ли факт пропуска с целевой переменной — если да, это ценный признак.

## Стратегии заполнения

### Числовые признаки
- **Медиана** — устойчива к выбросам, хороший дефолт.
- **Среднее** — если распределение симметричное.
- **Константа** (0 или -1) — когда пропуск имеет особый смысл.

### Категориальные признаки
- **Мода** (самое частое значение).
- **Новая категория** "unknown" — когда пропуск — это отдельная категория.

### Продвинутые методы
- **KNN Imputer** — заполняет по ближайшим соседям. Медленный на больших данных.
- **Iterative Imputer** — моделирует каждый признак по остальным.

## Создание индикатора пропуска
Для каждого столбца с пропусками можно создать бинарный признак `{column}_is_missing`. Деревья могут использовать этот признак для лучшего разбиения.
