# Советы по работе с данными Airbnb

## Особенности датасета
Данные объявлений краткосрочной аренды. Целевая переменная — количество дней доступности (availability_365).

## Полезные признаки

### Географические
- Широта и долгота можно использовать напрямую — деревья справятся.
- **Расстояние до центра Манхэттена** (Times Square: 40.758, -73.9855).
- **Кластеризация** координат (KMeans на lat/lon) — районы автоматически.

### Ценовые
- **Цена** (sum) — сильно коррелирует с доступностью и типом жилья.
- **Логарифм цены** — нормализует распределение.
- **Цена относительно района** — z-score цены внутри location.

### Хостовые
- **total_host** — количество объявлений хоста. Профессиональные хосты (>5 объявлений) управляют доступностью иначе.
- **Флаг профессионального хоста**: total_host > 5.

### Обзоры
- **amt_reviews** — количество обзоров, индикатор популярности.
- **avg_reviews** — средний рейтинг, может быть NaN если нет обзоров.
- **days_since_last_review** — давность последнего обзора, показывает активность.

### Тип жилья
- **type_house** — 3 категории: Entire home/apt, Private room, Shared room. Сильно влияет на доступность.

### Минимальные дни
- **min_days** — минимальное количество дней аренды. Высокие значения (30+) указывают на долгосрочную аренду.

## Распространённые проблемы
- Пропуски в last_dt и avg_reviews — обычно у новых объявлений без обзоров.
- Выбросы в цене (sum) — могут быть ошибочные значения.
- Название (name) — можно извлечь длину, наличие ключевых слов, но обычно малоинформативно.
